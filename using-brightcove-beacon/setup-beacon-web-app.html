Introduction

Note that this topic applies only if you purchased the Beacon Web App.

Like any web app, the Beacon Web App requires a structured collection of files available to the internet, security certificates to guarantee the authenticity of the sites, and fronting by a CDN to make the pages readily available to users anywhere in the world. Brightcove manages all this for you, but there are two steps you need to perform to enable the Beacon Web App to be associated with your domain.

Steps

Domain validation

  1. To validate your domain name, Brightcove will provide you with a CNAME record correspond to each variation of the domain name (for example, you may want to all both www.myplace.com and myplace.com).
  2. These CNAME records must be added to your DNS database. Contact your your DNS provider for instructions on how to do this.
  3. These DNS entries will be used to validate that you own the domain name, and it will then be attached to a security certificate for Beacon Web App.

Redirecting the domain name to the Beacon Web App

  1. Once the domain name validation is complete, Brightcove will provide you another DNS entry that will route requests for your domain to the internal address of your Beacon Web App.
  2. Add this entry to your DNS database also.
  3. Once the update to the DNS database is propagated across the internet (this may take up to 24 hours), your Beacon Web App will be available.
