Introductions

For iOS and Android apps you can send notifications using Google Firebase. This document will walk you through the process. The configuration of the iOS and/or Android apps to enable this functionality is done by Brightcove.

Sending a notification

The implmentation tasks to send a notification are detailed in the remainder of this document. Those steps are:

  1. >Configure the notification

Configure the notification

  1. Navigate to the Firebase Console and select your project.
  2. In the left navigation select:
    1. Engage
    2. Cloud Messaging
  1. Click Send your first message.
  2. Supply the Notification title and Notification text. Note that you can see a preview on the bottom right of the screen.
  3. Add an optional notification image, if desired.
  4. Add an optional notification name, if desired.
  5. If a development device is configured, you can test the message.
  6. Click Next.

Define the target

When you define the target you configure which apps should receive the notification and also define any further restrictions on whom should receive the notification.

  1. Select the app or apps to which you wish to send the notification.
  2. Add any restrictions for each app to which you are sending the notification.
  3. Click Next.

Schedule when the notification will be sent.

You have the option to send the notification immediately or have it sent repeatedly.

  1. Choose when you will send the notification.
    Do not choose the send now option until you have configured the video asset to which the notification will links. This process is detailed in the next section.

Add options for the video asset to which the notification will link

You will often link to a video asset that will be displayed when the viewer clicks a link in your notification. The video assets is defined in the Addition options section. The options from which you can choose are defined in this table:

Note the following:

  • The season_id and series_id are optional and only used in case of asset_type is an episode.
  • The series_id is also used if the asset_type is a season.
  • If you have the Explore option disabled and user has not authenticated yet, when the the app opens from a notification and the user authenticates the details page for the video asset is displayed to the user.
  • When a asset is opened from a push notification, the workflow is the same as opening an asset from a carousel regarding:
    • Entitlements checks and buttons displayed
    • Continuous Play (applicable when the asset in the notification is a series/seasons)
