Overview

There are two types of purchases that can be done with Beacon, those being:

  • Purchases done outside Beacon - For those purchases a webhook/API is supported that allows external systems to notify Beacon of any purchases related events. This enables the Beacon entitlement subsystem to properly make assets available to viewers.
  • Purchases executed through Beacon - Of course, the Beacon entitlement subsystem still makes assets available to viewers. Also available are notifications that are sent to the seller of the asset. This permits the asset supplier to display to the purchaser, via the external store, that a purchase has been made.

How notifications are implemented

Here is a high-level diagram of the process:

\For purchases that are done outside of Beacon, Beacon already supports a webhook/API that allows external systems to notify Beacon of any purchases related events so that the Beacon entitlement subsystem can then properly make assets available to viewers. When purchases are executed through Beacon, the same entitlement subsystem is updated but we now need to also notify the external system so that they know that a specific user has already purchased a specific asset and either grant additional entitlements that come from the purchase (like merchandise, etc) and can also inform the user in the external store that the asset is purchased (preventing double purchase). Beacon will report all purchase related events, including new subscriptions, renewals, cancellations, pauses, holds and resumes of subscriptions.
