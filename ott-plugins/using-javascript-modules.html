Basics

Whether you are going to use modules or another plan, there are some basics you need to understand before you start writing code. The first concept to understand is that Beacon plugins are implemented using an event based framework. Essential to the process is using window.postMessage which provides easy cross domain functionality. The following describe some of the fundamentals of implementing your plugins.

Dispatching an event

The skeleton code for dispatching an event is:

window.postMessage({
  event: "eventType",
  data: {
    key1: value1,
    key2: value2,
    ...
  }
}, window.location.origin)

where

  • eventType: The event being posted, for example enablePlayerSidePanel
  • data: Data passed for use

The following is an example that adds a button to a details page, passing as data for:

  • The text to appear on the button
  • The icon on the button
  • An ID for later use in a handler
window.postMessage({
  event: 'detailsPageAddCustomButton',
  data: {
    title: 'Test Button',
    font_awesome_icon: 'fa fa-info-circle',
    element_id: 'TEST_BTN_ID'
  }
}, window.location.origin);
