Process overview

A high-level look at the process of using OAuth with Brightcove Beacon is:

  1. Create an Auth0 API.
  2. Create an Auth0 single page application for your web application, if used. Once created, you will need to send the associated Client ID to the Beacon web app team to get the corresponding Allowed Callback URLs.
  3. Create an Auth0 native application for all other mobile and TV application platforms you are using with Beacon. For each platform, once the applications are created you will need to send the associated Client ID to the specific Beacon platform team to get the corresponding Allowed Callback URLs.
  4. Enter the returned URLs from each platform team into the corresponding Allowed Callback URLs form field for each application: web, mobile and TV.
  5. From one of the applications, copy the OpenID Configuration URL and browse it.
  6. Use values from the browsed page and enter required values into the Auth Providers form in Beacon Classic.
