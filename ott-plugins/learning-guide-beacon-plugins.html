Introduction

Beacon plugins provide customization of your web apps not available through the normal page creation process in Beacon Studio. For instance, this 8 second video shows the side panel opening and closing under viewer control. What, where, how, why

This side panel can be populated with data about the video playing, or maybe if it is a live video you can place a chat application in the side panel for viewer comments and discussion. The side panel is just one element available only through Beacon plugins.

The plugin functionality is controlled with JavaScript and CSS. In Beacon Classic you point to one JavaScript file and one CSS file that are loaded at app startup. Here is a screenshot of Beacon Classic of where that data is entered:

The JavaScript file must be called index.js and the optional CSS file must be called index.css.

Available elements

Element Functionality
Page events and data Listen for page events like page loading, unloading and changing.
User events and data Listen for user events like sign in, register and sign out.
Player events and data Listen for events for side panel behavior.
Custom buttons events and data Add and remove buttons and listen for click events.
Custom <div>s ID selectors Place custom content in <div>s.
Custom pages Create a custom page in the Beacon Studio then populate with dynamically built HTML of your choice.
